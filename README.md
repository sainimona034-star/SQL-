📊 SQL Mini Project – E-commerce Analysis
📌 Overview

This project analyzes an e-commerce database using SQL to understand customers, products, orders, and sales.

🧠 Queries Used
🔹 Basic Queries
* Select customer and product data
* Filter data using WHERE
* Sort using ORDER BY

🔹 Intermediate Queries
* Use DISTINCT, BETWEEN, LIKE
* Create calculated columns (price × quantity)
* Handle missing data (NULL)

🔹 Aggregations
* COUNT() → total orders
* SUM() → total revenue
* AVG() → average order value
* GROUP BY → category/customer analysis

🔹 Joins
* INNER JOIN → orders with customers
* LEFT JOIN → all customers with/without orders
* Combine multiple tables

🔹 Subqueries
* Products above average price
* Customers with/without orders
* Highest order per customer

🔹 Set Operations
* UNION → combine results
* INTERSECT → common data

📈 Key Insights
* Few customers generate most revenue
* Some products are never sold
* High-priced products sell less
* Some items go out of stock
* Orders vary by day
* Certain payment methods are preferred

🚀 Impact
* Better customer targeting
* Improved inventory planning
* Increased sales opportunities
* Identification of data issues

💡 Recommendations
* Restock fast-selling products
* Remove or improve low-demand items
* Offer discounts to inactive customers
* Focus on high-value customers
* Fix missing or incorrect data
* Track sales using dashboards

🏁 Conclusion
This project shows how SQL can be used to analyze data and generate useful business insights.
